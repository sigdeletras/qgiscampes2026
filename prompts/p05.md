# Deepseek

Configurar VS Code para desarrollar plugins de QGIS con Claude es un proceso que te dará una base sólida. Aquí tienes la guía paso a paso para principiantes, dividida en dos partes: primero, la comunicación con Claude dentro del editor; segundo, y más importante, la configuración para que VS Code entienda a QGIS y puedas depurar tu código.

### 1. Tu Nuevo Compañero de IA: Claude en VS Code

La forma más directa de tener a Claude disponible en tu editor es usar la extensión oficial. Te ayudará a generar código, explicarte conceptos o corregir errores sin salir de VS Code .

**Paso a paso:**

1.  **Instala la extensión**: Busca "**Claude Code**" de Anthropic en el Marketplace de VS Code e instálala. La versión 1.98.0 o superior de VS Code es necesaria .
2.  **Configura tu acceso a la IA**: Es casi seguro que querrás usar tu propia clave de API de Anthropic para conectarte a Claude. Un método sencillo es configurar una variable de entorno:
    *   **En Windows (PowerShell)**: `$Env:ANTHROPIC_API_KEY="tu-api-key-aqui"`
    *   **En macOS/Linux**: `export ANTHROPIC_API_KEY="tu-api-key-aqui"`
    *   Para que la variable sea permanente, agrégala a tu perfil de shell (`.bashrc`, `.zshrc`).
3.  **Empieza a usarlo**: Abre la barra lateral de Claude (el ícono de una chispa) y empieza a conversar. Pídele, por ejemplo: *"Genera la estructura base de un plugin QGIS que se llame 'mi_plugin_catastro'."*

---

### 2. El Corazón del Asunto: Configurar VS Code para "Hablar" con QGIS

Esta es la parte más técnica y la que hará que tu flujo de trabajo sea profesional. El objetivo es que VS Code entienda las librerías de QGIS, te ofrezca autocompletado (IntelliSense) y, lo más importante, puedas depurar tu código línea por línea .

#### Requisitos previos
*   **QGIS instalado**: Asumiremos una instalación estándar de OSGeo4W (ej. `C:\OSGeo4W` en Windows).
*   **Python**: El que viene incluido con QGIS.

#### Paso 1 (El más fácil): Crea un entorno virtual "clon" de QGIS

El mayor desafío es hacer que VS Code reconozca las librerías de QGIS. La herramienta más moderna y sencilla para esto es `qgis-venv-creator` .

1.  Abre una terminal (PowerShell en Windows) e instala la herramienta:
    ```bash
    pipx install qgis-venv-creator
    ```
2.  Navega a la carpeta raíz de tu plugin.
3.  Ejecuta el comando mágico:
    ```bash
    create-qgis-venv --venv-name ".venv"
    ```
    Esto escaneará tu sistema, encontrará tu instalación de QGIS y creará una carpeta `.venv` con todas las referencias necesarias. ¡VS Code la detectará automáticamente! 

#### Paso 2: El Arte de la Depuración Remota (Aquí la magia sucede)

Para poder poner puntos de interrupción y "ver" qué hace tu código dentro de QGIS, usaremos el plugin **QGIS DevTools** .

1.  **En QGIS**: Ve a `Plugins -> Administrar e Instalar Plugins...`. Busca e instala "**QGIS DevTools**".
2.  **En QGIS**: Una vez instalado, actívalo desde `Plugins -> QGIS DevTools -> Activar Depuración`. Por defecto, se conectará al puerto `5678`.
3.  **En VS Code**: Crea un archivo `.vscode/launch.json` en la raíz de tu proyecto con esta configuración:

    ```json
    {
        "version": "0.2.0",
        "configurations": [
            {
                "name": "Python: Conectar a QGIS",
                "type": "debugpy",
                "request": "attach",
                "connect": {
                    "host": "localhost",
                    "port": 5678
                },
                "pathMappings": [
                    {
                        "localRoot": "${workspaceFolder}",
                        // !!! AQUÍ VA LA RUTA A LA CARPETA DE PLUGINS DE QGIS !!!
                        "remoteRoot": "C:/Users/TU_USUARIO/AppData/Roaming/QGIS/QGIS3/profiles/default/python/plugins/mi_plugin_catastro"
                    }
                ]
            }
        ]
    }
    ```
4.  **En VS Code**: Ve a la vista "Run and Debug" (Ctrl+Shift+D) y selecciona la configuración "**Python: Conectar a QGIS**". Luego presiona "Start Debugging" (F5). Verás un mensaje en la barra inferior de VS Code indicando que está a la escucha.
5.  **¡A Depurar!**: Abre QGIS, ejecuta tu plugin. Si tienes un punto de interrupción (haciendo clic a la izquierda del número de línea en VS Code), ¡la ejecución se pausará y podrás inspeccionar variables! 

**Nota**: Para que la depuración funcione sin problemas, te sugiero crear un enlace simbólico (symlink) de la carpeta de tu proyecto a la carpeta de plugins de QGIS. Así, QGIS ejecuta el código desde tu carpeta de trabajo y VS Code puede depurarlo directamente.

## 🔌 Extensiones de utilidad imprescindibles para tu proyecto

Para desarrollar un plugin de QGIS no basta con la IA; necesitas que VS Code entienda el entorno de QGIS (que usa un Python interno de la aplicación) y las interfaces de Qt. Instala estas 4 extensiones:

### 1. Python (de Microsoft)

* **Para qué sirve:** Es obligatoria. Te da resaltado de sintaxis, detección de errores mientras escribes y formateo de código.
* **Tip de principiante:** Cuando instales QGIS, este viene con su propio ejecutable de Python que incluye las librerías `qgis.core` y `qgis.gui`. En VS Code, haz clic abajo a la derecha donde pone la versión de Python ("Select Interpreter") y apunta al Python que está dentro de la carpeta de instalación de QGIS. Así VS Code no te marcará como error los `import qgis`.

### 2. PYQGIS 3 Autocomplete

* **Para qué sirve:** La API de QGIS es gigantesca y difícil de recordar. Esta extensión le enseña a VS Code todos los módulos de PyQGIS para que el autocompletado te sugiera los métodos correctos de las capas, el proyecto y los mapas mientras escribes.

### 3. Qt Configuring and Tools (o PYQT Integration)

* **Para qué sirve:** Como vas a diseñar tu interfaz con **Qt Designer** (que genera archivos `.ui`), esta extensión te permite hacer clic derecho sobre el archivo `.ui` en VS Code y abrirlo directamente en el diseñador visual, o previsualizar cómo va quedando la ventana sin salir del editor.

### 4. Ruff (o Black Formatter)

* **Para qué sirve:** Mantener el código limpio. Autocorrige los espacios, sangrías y formateo de tu código Python cada vez que guardas el archivo (`Ctrl+S`). Es la herramienta perfecta para limpiar el código que te escupe la IA y asegurar que se vea profesional.

## 💡 Dos trucos para usar Claude en el día a día

* **Usa el comando `@`:** En el chat de la extensión, puedes escribir `@` seguido del nombre de un archivo (por ejemplo, `@metadata.txt` o `@catastro_plugin.py`). Esto le "pasa" el archivo exacto a Claude para que trabaje sobre él sin tener que copiar y pegar texto.

Para tu plugin, empieza en Normal mode siempre.

### Configuración de `settings.json` recomendada

Abre la paleta de comandos (`Ctrl+Shift+P`) → `"Open User Settings (JSON)"` y añade:

```json
{
  // Python
  "python.analysis.typeCheckingMode": "basic",
  "python.analysis.autoImportCompletions": true,

  // Ruff como formateador por defecto
  "[python]": {
    "editor.defaultFormatter": "charliermarsh.ruff",
    "editor.formatOnSave": true,
    "editor.codeActionsOnSave": {
      "source.organizeImports": "explicit"
    }
  },

  // Editor general
  "editor.rulers": [88],
  "editor.bracketPairColorization.enabled": true,
  "files.trimTrailingWhitespace": true,
  "files.insertFinalNewline": true
}
```

La línea `rulers: [88]` muestra una guía visual en la columna 88, que es el límite de línea que usa Ruff por defecto. Útil para no escribir líneas demasiado largas.

---

### Un fichero `ruff.toml` mínimo para el proyecto

Colócalo en la raíz del plugin:

```toml
line-length = 88
target-version = "py39"

[lint]
select = ["E", "F", "W", "I"]   # errores, pyflakes, warnings, imports
ignore = ["E501"]                # las líneas largas ya las controla el ruler
```

Con esto, cada vez que guardes un fichero Python, Ruff lo formatea y ordena los imports automáticamente, sin que tengas que pensar en ello.
