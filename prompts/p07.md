### El archivo más importante: `memorias_de_aprendizaje.md`

Es un diario donde documentas **cada interacción significativa con la IA**. Usa este formato:

```markdown
# Diario de Desarrollo - Plugin Catastro INSPIRE

## [2025-01-15] Aprendiendo a parsear ATOM con namespaces

**Mi pregunta a Claude:**
"¿Cómo extraigo la URL de descarga de un feed ATOM si tiene namespaces?"

**Su respuesta (resumida):**
Claude me explicó que debo usar `.findall('{http://www.w3.org/2005/Atom}entry')` 
y me dio un ejemplo con `nsmap`.

**Lo que aprendí:**
- Los namespaces en XML no son opcionales. Ignorarlos = no encontrar nada.
- En `xml.etree`, hay que declarar el namespace completo en cada búsqueda.
- Alternativa: usar `nsmap` y pasar prefijos.

**Código que funciona (con mi comentario):**
```python
# La clave: registrar el namespace ANTES de buscar
ns = {'atom': 'http://www.w3.org/2005/Atom'}
entries = root.findall('atom:entry', ns)  # ¡así sí funciona!
```

**Error que cometí:**
Intenté usar `findall('entry')` sin namespace. Me devolvía lista vacía 
y no entendía por qué. Claude me explicó que el XML declara xmlns por defecto.

**Próximo paso:**
Mañana voy a aplicar esto para filtrar por municipio.
```

**Por qué funciona**: Cuando olvides esto dentro de 2 semanas, no necesitas preguntar a Claude otra vez. Tu propio diario te lo explica en tu idioma y con tu contexto.

### Técnica 2: "Código con historial de aprendizaje"

Pídele que **incruste comentarios de aprendizaje** en el código:

```python
def parse_atom_feed(feed_url: str, municipio: str) -> Optional[str]:
    """
    [DOCUMENTACIÓN NORMAL AQUÍ]
    """
    # [APRENDIZAJE 2025-01-15] Los namespaces son OBLIGATORIOS.
    # Me costó 2 horas entender que sin el 'atom:' prefijo, findall devuelve vacío.
    # Alternativa: si usas lxml, puedes hacer xpath con '//*[local-name()="entry"]'
    
    import requests
    import xml.etree.ElementTree as ET
    
    # [POR QUÉ] Usamos ET en lugar de lxml porque viene con Python estándar.
    # No queremos dependencias extra para este plugin.
    
    response = requests.get(feed_url, timeout=30)
    response.raise_for_status()  # [APRENDIZAJE] Sin esto, errores silenciosos
    
    # [ERROR COMÚN] Si el XML tiene BOM (Byte Order Mark), ET se rompe.
    # Solución: response.content (bytes) en lugar de response.text (string)
    root = ET.fromstring(response.content)
    
    # [POR QUÉ ESTO ES CONFUSO] En XML con namespaces, 'entry' no existe.
    # Necesitas el namespace completo o usar el prefijo registrado.
    ns = {'atom': 'http://www.w3.org/2005/Atom'}
    
    for entry in root.findall('atom:entry', ns):
        # [PARA RECORDAR] El título a veces viene con acentos raros.
        # Normalizar con unicodedata si falla la comparación.
        title = entry.find('atom:title', ns).text
        if municipio.lower() in title.lower():
            # [MEJORA POSIBLE] Buscar el link con rel="enclosure"
            link = entry.find('atom:link[@rel="enclosure"]', ns)
            if link is not None:
                return link.get('href')
    
    return None  # [CUIDADO] None vs raise exception según el caso
```

## El principio clave

Documentar con IA no es pedirle que escriba la documentación por ti. Es usarla como **interlocutor que te obliga a entender antes de avanzar**. La diferencia:

- ❌ *"Documenta esta función"* → obtienes texto, no aprendes nada

## Qué documentar y cómo, con ejemplos concretos

### 1. Docstrings: el hábito más rentable

Instala `autoDocstring` en VS Code. Al escribir `"""` debajo de una función, genera el esqueleto automáticamente. Luego pide a Claude que lo rellene *explicándote* cada parte:

**Prompt útil:**
> *"Tengo esta función. Antes de escribir el docstring, explícame qué hace cada parámetro y qué puede salir mal. Luego escribe el docstring en español, estilo Google."*

```python
def get_municipality_entries(province_feed_url: str) -> list[dict]:
    """
    Obtiene la lista de municipios disponibles desde un feed ATOM provincial.

    Args:
        province_feed_url: URL del feed ATOM de una provincia concreta.
            Ejemplo: 'https://www.catastro.hacienda.gob.es/INSPIRE/.../ES.SDGC.CP.atom.xml'

    Returns:
        Lista de diccionarios con claves 'name' y 'download_url' por municipio.
        Ejemplo: [{'name': 'Sabadell', 'download_url': 'https://...'}]

    Raises:
        requests.Timeout: Si el servidor no responde en 30 segundos.
        ValueError: Si el feed no contiene entradas válidas.
    """
```

El valor de aprendizaje está en la conversación previa al docstring, no en el texto final.

### 2. Un `README.md` que crece con el proyecto

No lo escribas al final. Mantenlo vivo desde el día uno como diario técnico. Estructura sugerida:

## Qué hace
Descarga datos INSPIRE (parcelas, direcciones, edificios) por municipio
desde los feeds ATOM de Catastro y los carga como capas en QGIS.

## Arquitectura
- `atom_parser.py` — parsea el feed en dos niveles (provincia → municipio)
- `downloader.py` — descarga el ZIP y extrae el GML
- `layer_loader.py` — carga la capa en QGIS con iface
- `dialog.py` — UI: selector de provincia, municipio y tipo de dato

## Decisiones técnicas y por qué
### Por qué dos niveles de feed
El feed raíz lista provincias, no municipios directamente. Hay que
hacer una segunda petición al feed provincial para obtener los municipios.
Aprendí esto al inspeccionar el XML manualmente.

### Por qué QgsTask para la descarga
La primera versión bloqueaba la interfaz durante la descarga. QgsTask
ejecuta la operación en segundo plano. Referencia: [PyQGIS Cookbook - Tasks]

## Problemas encontrados y soluciones
| Problema | Causa | Solución |
|---|---|---|
| La capa no se mostraba | Ruta con espacios en Windows | Usar QgsProcessingUtils.tempFolder() |
| Feed devuelve 403 | Falta User-Agent en la petición | Añadir headers={'User-Agent': 'Mozilla/5.0'} |

## Qué aprendí en cada módulo
...
```

**Prompt útil para mantenerlo:**

> *"Añade comentarios inline a este bloque explicando por qué se hace así en QGIS, no solo qué hace. Especialmente en las partes que usan la API de PyQGIS."*

```python
# QgsVectorLayer necesita el proveedor "ogr" para ficheros GML locales.
# Sin este tercer argumento, QGIS no sabe cómo leer el fichero.
layer = QgsVectorLayer(gml_path, layer_name, "ogr")

# isValid() comprueba si QGIS pudo parsear el fichero correctamente.
# Un layer inválido no lanza excepción, simplemente no carga — hay que verificarlo.
if not layer.isValid():
    raise ValueError(f"QGIS no pudo cargar la capa desde {gml_path}")

# addMapLayer() registra la capa en el proyecto actual.
# Sin esto, la capa existe en memoria pero no aparece en el panel de capas.
QgsProject.instance().addMapLayer(layer)
```
