## Puntos clave (viabilidad)

| Aspecto | Dificultad | Notas para ti (Python básico + web) |
|--------|-----------|--------------------------------------|
| **Leer ATOM** | Baja | Es un XML/RSS. Con `requests` + `xml.etree` lo haces. Similar a consumir APIs web. |
| **Filtrar por municipio** | Media | Los ATOM dan URLs de descarga por provincia y municipio (código INSPIRE). Tendrás que parsear el XML y buscar el entry con el `dc:spatial` que coincide con el nombre/código municipal. |
| **Descargar GML** | Baja | Una vez tengas la URL de descarga (dentro del `<link>` del entry), usas `requests` para bajar el archivo `.gml`. |
| **Cargar GML en QGIS** | Baja/Media | QGIS puede abrir GML directamente con `iface.addVectorLayer(ruta_gml, "nombre", "ogr")`. Aunque a veces el CRS o la geometría multienlace requiere atención. |
| **Manejo de volumen** | Media | Las parcelas de un municipio mediano pueden ser >200 MB. El plugin puede quedarse "congelado" si no usas barra de progreso (`QProgressDialog`) o hilos (`QThread`). Pero para empezar, hazlo simple. |

## Recomendaciones para tu plugin
1.  **Empieza por dirección o edificios** (son capas más ligeras que parcelas).
2.  Usa `QgsNetworkAccessManager` en lugar de `requests` si quieres respetar la configuración de proxy de QGIS.
3.  Para filtrar municipio: el usuario escribe el nombre, tú buscas en el ATOM la coincidencia con `<title>` o `<subtitle>` (a veces está en `<author>`). Ten a mano un CSV con la relación nombre_municipio → código INSPIRE oficial.
4.  **Cuidado**: La primera descarga de un municipio tarda (el servidor genera el GML sobre la marcha). Implementa un indicador de "descargando..." para no confundir al usuario.

## Valoración de viabilidad

**Veredicto: totalmente viable, y es un proyecto ideal para un primer plugin.**

### Lo que juega a tu favor

La API ATOM de Catastro es pública, sin autenticación, y devuelve XML estándar con enlaces a descargas GML/ZIP por municipio. El flujo es predecible: parsear el feed → localizar el enlace del municipio → descargar el ZIP → extraer el GML → cargar con `iface.addVectorLayer()`. Nada de esto requiere conocimientos avanzados de PyQGIS.

Tu background web es una ventaja real: parsear XML con `xml.etree.ElementTree`, hacer peticiones HTTP con `urllib` o `requests`, y manipular rutas de ficheros son cosas que ya conoces en otro lenguaje. La traducción a Python es directa.

### Las partes con algo más de fricción

**Estructura del plugin** — QGIS exige unos ficheros mínimos (`__init__.py`, `metadata.txt`, el fichero principal) con una firma concreta. No es complejo, pero hay que conocerlo. La herramienta **Plugin Builder** (disponible en el gestor de complementos de QGIS) te genera ese esqueleto en segundos y elimina esta barrera.

**El feed ATOM tiene dos niveles** — el feed raíz lista provincias, y cada provincia tiene su propio sub-feed con los municipios. Tendrás que hacer dos peticiones HTTP, no una. Nada grave, pero hay que tenerlo en cuenta al diseñar la lógica.

**La descarga es un ZIP** — el GML viene comprimido. `zipfile` (módulo estándar de Python) lo resuelve en 3 líneas.

**Hilo principal de QGIS** — si haces la descarga en el hilo principal, la interfaz se congela. Para un primer plugin es aceptable; si quieres hacerlo bien desde el principio, `QgsTask` o `QThread` lo soluciona, aunque añade complejidad.
